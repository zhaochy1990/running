# STRIDE Mobile App — 功能规格

> 重写版手机 App 的功能规格文档。聚焦训练故事线。

---

## 主线故事一句话总结

> 注册 → 绑表（必须）→ 同步历史 → 基础信息 → 主页（看到历史活动 + AI 点评）→ 主动设训练目标 → 同步 3 年历史 → 生成长期总纲 → 聊天式 review 总纲 → 基于总纲生成本周计划 → 推手表 → 每天打开看今日课 → 跑完看 AI 点评 + 填 RPE → 周日看复盘 + 滚动生成下周计划 → 循环

**关键设计原则**：
- Onboarding 后，App **立即有价值**——用户能看到自己过去 90 天的活动 + AI 点评
- 训练计划变成**主动触发**的高级功能，不强推
- 降低注册到首次"啊哈时刻"的距离
- **手表是 App 的前提**：所有核心功能都依赖手表数据，新用户必须完成绑定才能进主流程

# 模块 A：身份认证

## A1. 启动
- 冷启动后根据 token 状态自动路由：
  - 有 valid token → 主页
  - token 过期但有 refresh → 静默刷新 → 主页
  - 无 token → 登录
- 网络异常 → 跳转登录并提示

## A2. 登录
- 支持邮箱 + 密码登录
- 错误反馈：账号密码错误 / 网络异常分别给出不同文案
- 入口跳转：登录成功 → 主页；新用户 → 注册

## A3. 注册
- 邮箱 + 密码 + 邀请码注册
- 实时校验：邮箱格式、密码强度（≥8 位）、两次密码一致、邀请码非空、协议勾选
- 注册成功 → 进入新手引导

# 模块 B：新手引导

引导流程共 4 步，**手表绑定是必需步骤**，引导完成后直接进主页（不强制生成训练计划）。

## B1. 选择手表品牌
- 支持品牌：COROS（v1 主推）、Garmin（v1.1 即将支持）
- **不允许跳过**：所有核心功能都依赖手表数据，必须绑定一款手表才能进入主流程

## B2. 绑定手表账号
- 输入 COROS 邮箱 + 密码
- 自动检测区域（全球 / 中国 / 欧洲），允许手动覆盖
- 明确告知用户：仅读取训练数据，不修改手表设置
- 错误处理：区域不匹配 → 提示切换；密码错 → 提示核对 COROS 登录密码
- 绑定失败不可越过此步，必须成功才能继续

## B3. 首次同步
- 拉取过去 90 天活动 + 健康数据到本地
- 同步过程不可中断；失败可重试
- 实时反馈进度：已同步活动数 / 健康数据天数

## B4. 基础信息采集
- 性别、出生年、身高、体重
- 静息心率（可从手表数据自动填）
- 最大心率（可从手表数据自动填，或按 220-age 公式自动填）
- 完成 → 进入主页

## B5. 未绑表用户的处理
- 已注册但当时没绑表的旧用户、或解绑了手表的用户进入 App 时：
  - 主页 / 训练 Tab / 身体指标等所有核心功能都不可用
  - 全屏引导卡 + 「立即绑定手表」CTA → 跳转 B1
  - 不允许通过任何方式跳过绑定回到主流程

# 模块 C：训练总纲

模块 C 围绕"训练总纲"这一核心概念组织。总纲是用户长期训练的"地图"，定义周期、阶段、关键里程碑，是后续单周生成的基础。

## C1. 训练目标采集
计划生成是用户主动触发的高级功能，入口包括主页 CTA 卡和训练 Tab 空状态按钮。触发后先收集训练目标：
- 目标类型：备赛 / 提升 PB / 减脂 / 健康跑 / 维持（单选）
- 选「备赛」时进一步采集：比赛日期、距离（5K/10K/半马/全马/越野）、目标完赛时间（可选）
- 每周可训练天数（3-6）
- 每周可训练时段（早 / 午 / 晚，多选）
- 力量训练意愿（愿意 / 不愿意 / 看情况）

## C2. 跑步背景采集（可选）
- 跑龄区间
- 当前周跑量区间
- 各距离 PB（5K/10K/半马/全马，可全空）
- 伤病史（多选 + 「暂无」）
- 可整体跳过，使用默认值

## C3. 数据准备
- 在生成前，先同步用户最近 3 年的跑步记录到本地（B3 首次同步只覆盖 90 天，更长历史按需在此触发）
- 数据量可能较大，需独立的进度页展示：已同步活动数 / 总数、当前年份段、进度百分比
- 同步过程不可中断；失败可重试；已同步部分可断点续传

## C4. 总纲生成
- 服务端基于历史数据 + 目标 + 背景，生成**长期训练总纲**（覆盖到比赛日 / 目标周期，划分基础期 / 进展期 / 赛前期 / 比赛 / 恢复等阶段）
- 总纲粒度：阶段定位、阶段周数、每阶段的训练重点 / 周量区间 / 关键课型 / 关键里程碑
- 整个生成过程可能耗时较长（分钟级），用户可以离开 App 或稍后再回来查看进度，无需一直停留
- 展示分阶段进度文案：读取历史数据 → 评估当前体能（CTL/ATL/TSB）→ 结合目标 → 规划周期阶段 → 输出总纲
- 生成在服务端后台进行，完成后通过推送通知召回用户

## C5. 总纲 review（聊天式）
- 展示总纲概览：周期时间轴 + 各阶段卡片 + 关键里程碑
- 用户用自然语言对话调整（如「基础期再多 2 周」「赛前期降低强度」「我 8 月份要出差，这周减量」）
- AI 返回修改后的总纲 diff（旧 → 新），用户可逐条采纳或拒绝
- 提供常用快捷指令气泡
- 用户确认后总纲落库，**不会自动生成任何单周计划**——单周计划永远在"上周训练结束 + 用户填反馈"之后才生成（详见 D1）
- 确认后直接进入 C2 总纲展示页；如本周尚无单周计划，主页会显示"立即生成本周计划"CTA 让用户主动触发首次生成

## C6. 总纲展示页

总纲落库后的常驻展示位，用户可随时查看长期训练地图。

### 展示内容
- **总体进度高亮**
  - 总纲覆盖周期（起止日期 / 总周数）
  - 当前所处阶段、当前周次（如「进展期 W3 / 共 6 周」）
  - 距下一个里程碑 / 比赛日的倒计时
  - 整体完成度（已过去周数 / 总周数）
- **阶段时间轴**
  - 横向时间轴：基础期 → 进展期 → 赛前期 → 比赛 → 恢复（具体阶段由总纲决定）
  - 高亮"当前阶段"，已完成阶段、未来阶段视觉区分
- **阶段详情卡（每阶段一张）**
  - 阶段名 + 起止日期 + 周数
  - 训练重点（如「打基础有氧、提升周量到 50K」）
  - 周量区间、关键课型（如「长距离 + 节奏跑」）
  - 阶段里程碑（如有）
- **里程碑清单**
  - 类型：测试赛、关键长距离、强度测试、目标赛事等
  - 日期、所属阶段、目标说明（如「10 月 15 日：30K 节奏跑测试，配速 4'45/km」）
  - 已完成的里程碑展示实际完成情况

### 操作入口
- 进入任一阶段卡 → 查看该阶段所有周的概览
- 「与 AI 教练对话调整总纲」入口 → 进入 C7 总纲对话调整
- 备赛模式下顶部固定显示距比赛日倒计时

## C7. 总纲对话调整（与 AI 教练沟通）

总纲不是初次生成后就一成不变的。用户在整个训练周期中可以随时进入此页，以聊天的方式与 AI 教练沟通，对总纲进行调整。

### 触发场景
- **目标变化**：调整赛季目标（如从「完赛」升级为「sub-330」），改变备赛距离
- **比赛日期变化**：比赛延期 / 提前 / 取消 / 新增另一场比赛
- **训练反馈驱动**：连续几周完成度不佳 / RPE 偏高 / TSB 长期负向，需要降低强度或调整阶段长度
- **伤病 / 身体状况**：受伤需减量或暂停、康复期重新规划、出差/工作期临时降量
- **生活节奏变化**：每周可训练天数变化、训练时段变化

### 交互
- 聊天式对话界面，顶部贴当前总纲的概览缩略图（阶段时间轴 + 关键里程碑）
- 用户用自然语言描述变化（如「比赛推迟到 12 月 20 日」「我膝盖最近不舒服，下个月想减量 30%」「我想把目标改成 sub-320」）
- AI 教练基于变化重新规划总纲，返回修改后的总纲 diff（旧 → 新）
  - diff 高亮：阶段增减、阶段长度变化、周量调整、里程碑增删 / 改期
- 用户可逐项采纳或拒绝；也可以继续追问「为什么这样调整」「有没有别的方案」
- 提供常用快捷指令气泡（如「降低强度一档」「比赛延期 4 周」「增加一场测试赛」）
- 用户最终确认后，新总纲落库，并自动影响后续单周生成
  - 已生成的本周计划：提示用户是否同步重新生成
  - 已推送到手表的训练：提示哪些会失效，需手动清理（参考多变量计划改选的处理逻辑）

### 入口
- 总纲展示页（C6）顶部「与 AI 教练对话调整总纲」按钮
- 个人中心 / 主页的「调整总纲」入口

## C8. 总纲调整历史
- 保留每次总纲调整的版本快照
- 用户可在总纲展示页查看「调整历史」时间线（什么时候、因为什么、改了什么）
- 支持对比任意两个版本的差异

# 模块 D：单周训练与日常主循环

模块 D 子模块按"时间顺序"排列：周计划生成 → 查看 → 训练 → 反馈 → 复盘。

## D1. 单周生成
- 基于已确认的总纲 + 用户当前状态（疲劳 / TSB / 上周完成情况 + 本周反馈），生成下一周计划
- **核心原则**：单周计划永远是"上周训练结束 + 用户填反馈"之后才生成。反馈是关键输入，**没有反馈不应该自动生成下周**（特殊情况下可降级到规则引擎用默认值，但 UI 应明确告知用户）
- 生成耗时较短（秒级），无需后台运行
- **触发方式**：
  - **自动触发**：每周日晚 24:00 自动生成下一周计划（前提：本周至少有 1 节训练已完成 + 用户填了至少 1 条反馈），完成后推送通知用户查看
  - **手动触发**：用户可在周日提前完成本周训练并填写反馈后，从主页 / 训练 Tab 主动触发下周生成（无需等到 24:00）
- **首次本周生成（C5 确认总纲后）**：走和后续相同的**手动触发**路径——总纲 confirm 不自动生成；主页显示"立即生成本周计划"CTA，用户点击后才生成。此时无上周反馈，规则引擎降级使用默认基础周量 + 用户的跑步背景

## D2. 周计划预览
- 展示本周定位（如「基础期 W1 · 控量适应」）和总览数据（周里程 / 总时长 / 力量次数）
- 7 天课表，每天显示主课、时长、距离、强度标签（E/M/T/I/R/休）
- 操作：调整计划 / 推送到手表
- 推送结果反馈：成功条数 / 失败可重试

## D3. 课时详情
- 课名、类型标签
- 距离、时长、配速区间、心率区间
- 执行要点
- 训前 / 训中 / 训后营养建议
- 力量课额外展示动作清单（动作 + 组×次 + 组间休息）

## D4. 周计划调整（聊天式）
- 用户用自然语言描述调整意图（如「周三换到周四」「降低强度」「增加一节力量」）
- AI 返回修改后的计划 diff（旧 → 新）
- 用户可逐条采纳或拒绝
- 提供常用快捷指令气泡

## D5. 主页
根据当前状态展示不同内容：
- **有训练计划**：今日课卡（开始训练 / 已完成 / 休息日）+ 状态环（疲劳 / TSB / 训练负荷比）+ 本周进度 + 下一节关键课预告
- **无训练计划**：生成计划 CTA + 最近活动列表 + AI 点评摘要 + 状态环
- 顶部入口：用户头像（→ 个人中心）/ 当日日期 / 手动同步

## D6. 训练前
- 课名、强度、距离、时长、配速、心率等关键参数
- 热身清单
- 训前营养提示
- 提示用户去手表上启动训练

## D7. 训练后反馈
- 触发：手表同步完成后推送
- 用户填写：RPE（1-10）、一句话感受（可选）、感受标签（如「腿酸」「状态好」「天气热」「心情差」）
- 提交后写入 `sport_note` + `feedback.md`

## D8. 活动详情
- GPS 轨迹地图
- 关键指标：距离、时长、配速、心率、卡路里、海拔
- 多视图切换：
  - **AI 点评**：GPT-4.1 自动生成的草稿，可重新生成；可触发 Claude 深度版
  - **分段**：分段表
  - **心率**：心率曲线 + 心率区间分布
  - **配速**：配速曲线
- 训练反馈区：已填则展示 RPE + 备注；未填则提示填写

## D9. 周复盘
- 触发：周日晚推送，或主动从训练 Tab 进入
- 内容：
  - 本周完成率、总里程、总时长
  - TSB 走势
  - 每节课完成情况 + AI 点评
  - 3 条关键洞察
  - 下周计划预览（已自动生成草稿，承接 D1）

# 模块 E：身体指标

参考 Web 端 HealthPage / AbilityPage 的能力，集中展示用户身体状态、训练负荷、能力评估、成绩历史。

## E1. 健康概览
- 顶部关键指标卡：静息心率（RHR）、睡眠 HRV、疲劳值（fatigue）、训练负荷状态
- 每个指标显示：当前值、相对基线偏离方向、简短解读（如「已恢复 / 正常 / 疲劳 / 高疲劳」）

## E2. 训练负荷（PMC）
- ATL（短期负荷）/ CTL（长期负荷）/ TSB（体能储备）三条曲线
- TSB 状态分区可视化（比赛就绪 / 过渡区 / 正常训练 / 过度负荷 / 减量过多）
- AI 解读卡（如「TSB 已连续 3 天 < -25，建议明天降量」）
- 时间范围切换：30 天 / 90 天 / 180 天

## E3. 趋势详情
- 多维度切换：疲劳 / HRV / RHR / 睡眠 / 训练负荷
- 时间范围：7 天 / 30 天 / 90 天
- 趋势图 + 当前值 + 7 日均 + 趋势方向
- Garmin 用户额外展示：睡眠详情 / Body Battery / 压力 / HRV 每日趋势

## E4. 能力雷达
- 4-Layer custom score 雷达图：耐力 / 速度 / 阈值 / VO2max / 经济性 / 新鲜度等多维能力
- 每个维度展示当前值 + 历史趋势 mini chart
- 维度解读：每项能力的当前定位（弱 / 中 / 强）+ 提升建议

## E5. 成绩预测
- 主距离（用户目标比赛对应距离）大字预测：当前预测完赛时间 + 配速
- 多距离对比：5K / 10K / 半马 / 全马 同时展示当前预测
- 数据来源标注：实时计算 / 快照
- VO2max 面板：当前 VO2max 估计、最近趋势、刷新入口
- 与目标差距：用户在 C1 设过目标完赛时间 → 展示「当前预测 vs 目标」差距条
- 历史预测曲线：随训练推进，预测时间的变化趋势（让用户看到自己在变强）

## E6. PB 记录
- 4 个距离（5K / 10K / 半马 / 全马）
- 每个距离展示：当前 PB、达成日期、历史趋势
- 自动从活动数据中检测、识别、更新 PB
- 破 PB 时主页弹庆祝动画

# 模块 F：营养

营养是训练的三大组成之一（运动 / 营养 / 恢复）。营养模块独立成页，让用户管理偏好、查看每日建议。

## F1. 营养偏好设置
- 是否启用营养建议（总开关）
- 饮食类型：无忌口 / 素食 / 清真 / 其他
- 过敏 / 忌口食物（多选 + 自定义）
