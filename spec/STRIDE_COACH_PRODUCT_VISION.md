# STRIDE Coach — 产品愿景 & 商业计划（对话即入口）

> 状态：Draft · brainstorm 产出 · 当 BP 对待 —— 产品设计 + 盈利模式
> 核心命题：把「与教练对话」做成产品的**主入口**，用户在一个对话框里表达任意跑步相关诉求，系统识别意图并路由到对应能力；围绕"**专业训练科学 × 个体数据规模 × 执行闭环**"建立护城河，用**订阅 + 导购分成**双引擎变现。

---

## 1. 问题与愿景

### 1.1 问题陈述

中国跑步人群达 **1.63 亿**（2024），路跑赛事年参与超 **700 万人次**且持续增长（市场规模详见 §1.4），但绝大多数跑者面对同一组痛点：

- **凭感觉训练**：不懂周期化、负荷管理，容易过度训练受伤，或长期原地踏步不进步。
- **数据用不起来**：手表传感器采集了大量心率、配速、负荷、HRV 数据，但跑者不知道**如何把这些数据转化成训练决策** —— 今天该练什么、强度多少、要不要恢复，全凭感觉。
- **专业指导难获得**：线下私教/跑团贵，且受通勤、时间、地域限制 —— 所在城市未必有合适水平的教练；线上训练营是群发计划，跟不上个体每日状态。
- **需求是整体的**：训练之外，还要操心吃什么、怎么恢复、伤了怎么办、选哪场比赛、买什么装备 —— 没有一个连贯的入口。

**这对两类跑者都成立**：① 有明确成绩目标的**备赛跑者**（冲 PB / 完赛）；② 只想科学健康跑步、减脂、长期维持的**健康跑用户**。两者都需要"科学、个性化、可负担、随时随地"的指导，而这是当下大众未被满足的需求。

### 1.2 机会与愿景

**AI + 个体数据**首次让"专业教练级"的个性化训练指导可以**规模化、可负担、随时随地** —— 不再受限于本地有没有好教练。

STRIDE 不只是"训练计划工具"，而是**跑者围绕训练核心的全程伴跑教练**：记得住你的身体状态、训练历史、目标，覆盖跑者从入坑到 PB（或从久坐到健康跑）的完整生命周期。

### 1.3 护城河

**真正的护城河不是数据本身**（手表厂商也有数据，但只存不算），也**不是单个公式**（PMC、VDOT、LTHR 都是公开的运动生理学，谁都能查）。真正可防御的，是在**对标 RQ 的工程基线（①）之上**，由 ②③④ 与数据规模长期咬合出的飞轮：

- **① 多设备数据归一化的工程基线（对标 RQ）**：把异构原始数据清洗、对齐、归一是一切分析的地基。**这块 RQ 已经做得很扎实，是我们要对齐看齐的工程标杆，不是我们的独有壁垒**；我们在这一维的真实增量，是在 RQ 的 COROS/Garmin/Apple 基础上**额外扩到华为 / 小米等国产表**（最终覆盖华为/小米/Apple/COROS/Garmin + 手机 GPS 全主流来源），吃下大陆主流设备用户。
- **② 大规模真实数据校准的算法精度**：公式公开，但个体基线（max_hr / LTHR / threshold pace / zones，`running_calibration`）、负荷模型（CTL·ATL·TSB）、能力快照与瓶颈诊断的**准确度取决于校准数据量与工程打磨** —— 越多真实跑者，基线越准，越难被追平。把"一堆原始手表数据"变成"可决策的训练洞察"。
- **③ 周期化执行闭环**：针对有**明确目标**的跑者（备赛 PB / 完赛，或减脂 / 健康维持），建立**阶段化、周期化**计划（备赛分 base/build/peak/taper；健康跑按渐进负荷编排），并**持续跟踪执行** —— 每周对照实际负荷/疲劳/反馈动态调整，形成 `规划 → 执行 → 跟踪 → 调整` 越用越贴合个体的闭环。这正是 Sigma（计划仍邀请制）、RQ（计划偏 VDOT 静态模板、且无执行追踪）都还浅的一段。
- **④ 中文训练学 / 运动医学知识库**：让伤病、营养、恢复敢答且可信，是长期积累、难以速成的资产。

四者咬合成**数据飞轮**：能跟它们咬合的能力扩展，价值都被放大 —— 同样一句"推荐跑鞋"，普通导购只能按销量排，STRIDE 能按你的步频、落地、当前周期阶段、周跑量、伤病史推荐适配的鞋，**更可能转化、客单价更高**（个体语境导购的转化提升幅度待早期 A/B 验证）。

**一句话定位**：一个记得住你身体状态、训练历史、目标赛事的对话式跑步教练，覆盖跑者**从入坑到 PB（或从久坐到可持续健康跑）**的完整生命周期。

**双重价值主张**：
- 对**跑者**：用 AI 把"专业教练级"的训练规划 + 数据分析 + 装备/比赛决策，降到月费可负担。
- 对**品牌/赛事**：用真实个体数据语境，把装备/报名/酒店精准推荐给"对的跑者"，比泛投广告 ROI 高。

### 1.4 市场规模（TAM / SAM / SOM）

> 三层漏斗：**TAM**＝理论天花板（赛道想象空间）；**SAM**＝我们这套产品/商业模式能服务的细分；**SOM**＝3 年内现实可拿下的份额。`TAM ⊃ SAM ⊃ SOM`，越往里越保守。所有口径均标注**假设**与**来源**。

| 层 | 口径 | 人群 / 规模 | 年化营收空间（假设）|
|---|---|---|---|
| **TAM** | 全体中国跑步人群 × 跑步相关年消费 | **1.63 亿** 跑步人口；跑步产业池 **>4000 亿元/年**（人均年消费 ~¥3362）| 订阅 + 导购两引擎理论可触达 **4000 亿+ 消费池** |
| **SAM** | 有手表/数据 + 系统训练或备赛意图 + 付费力的跑者 | **~1500–3000 万人**（跑步人群的 ~10–18%）| 纯订阅天花板 **~90–180 亿元/年**（按 Pro ¥588/年）；导购另计 |
| **SOM**（3 年）| SAM 中现实可获取的付费用户 | **~15–90 万付费用户**（SAM 渗透 1–3%）| 订阅 ARR **~0.7–4.2 亿元/年**（按折后有效 ¥470/年）+ 导购分成 |

**SAM 收窄逻辑**：1.63 亿跑步人口里，多数是偶发慢跑、不会为训练付费；STRIDE 的可服务人群需同时满足三个条件 —— ① **有手表/可同步数据**（马拉松赛事 84% 跑者用穿戴设备；中国智能手表年出货 >3500 万，运动表占 ~27%）；② **有系统训练或备赛意图**（2024 路跑赛事参与 704 万人次、同比 +16.5%，去重后数百万级严肃跑者）；③ **有付费力**（路跑人群七成月入过万、六成本科以上）。三者交集估 ~1500–3000 万人。

**SOM 渗透假设**：早期订阅型 AI 产品现实渗透取 SAM 的 1–3%。**Base case** ≈ 30 万付费用户 → 订阅 ARR ~1.4 亿元/年 + 导购；**Bull case** ≈ 90 万 → 订阅 ~4 亿元/年 + 导购。导购引擎（R2–R4）对免费用户同样变现，可在订阅之上叠加（GMV × CPS，详见 §7）。

> **数据来源**：跑步人口 1.63 亿、路跑 749 场/704.86 万人次（+16.5%）、跑者年均消费 ¥3361.8、收入/学历结构 —— 中国田径协会《2024 年中国路跑工作报告》（2025-03）及新华网/新浪财经转载；穿戴设备渗透 —— 公开行业市场报告（2025）。**人群交集、渗透率、ARPU 均为本文假设，需早期实测校准。**

## 2. 市场与竞品定位

| 类别 | 代表 | 模式 | STRIDE 的差异 |
|---|---|---|---|
| **国内/华语 AI·数据跑步（直接竞品）** | **Sigma**（超超世世科技·上海）/ **RQ 跑力**（RunningQuotient）| 订阅 | 唯一与我们**争夺同一批中国跑者**的两家；Sigma **记录审美先行**、RQ **数据分析先行**，**周期化对话闭环均浅**——详见下方深度分析 |
| **AI 跑步教练（全球标杆 · 间接）** | **Runna**（2025 被 Strava 收购）| 订阅 $19.99/月·$119.99/年 | 产品形态最强参考，但**不进中国**、不与我们正面争夺用户；核心短板:**不吃生理信号、无双向对话、伤病口碑差** |
| 综合运动社交平台 | Strava | 订阅 + 社交 | Athlete Intelligence 只**解释过去**(描述性)，非**规划未来**(处方性)；大陆受限 |
| 国内社交跑步 | 咕咚 / 悦跑圈 / Keep | 免费 + 广告 + 电商 + 社交 | 触达大、有本土支付/赛事，但**无严谨 AI 周期化教练**；STRIDE 用数据深度切高客单跑者 |
| 西方专业训练 | TrainingPeaks / Final Surge | 订阅（~$20/月）| 它们弱在**中国本地化（赛事/装备/支付/语言）+ 对话低门槛**；且非 AI 原生 |
| 手表自带 | Garmin Connect / COROS | 免费随表 | 数据全但**不解读、不规划、不对话** |
| **真人教练（线下）** | 跑团 / 私教 | 高价（¥/月数百-数千）| 个性化够，但**受通勤/时间限制、对大城市不友好；且依赖本地有没有同水平教练**——可达性差 |
| **线上训练营** | 各类微信训练营 | 中价（¥/期）| 可达，但多为**群发计划/统一指导，个性化不足**，跟不上个体每日状态 |

**竞争格局判断（谁才是"直接"竞品）**：是否服务中国市场，是判断直接竞争的分水岭——不进中国的产品不与我们争夺同一批用户，只能作为**产品形态参考与品类验证**。据此分两层：

- **直接竞品（同市场、同人群）**：**Sigma、RQ（跑力）** 等中文/华语圈 AI·数据跑步产品——下文重点拆解。
- **品类标杆 / 间接（形态参考，但不进中国）**：**Runna**（全球最强形态）、TrainAsONE、Humango、Coopah 等——它们验证了"AI 教练"模式成立，但无中文、无本土设备/支付/赛事，**不与 STRIDE 正面争夺中国跑者**。

下文先深拆中国侧两家直接竞品，再把欧美玩家作为形态标杆快速扫描。

### Runna 深度分析（全球品类标杆 · 间接竞品）

Runna 是品类标杆 AI 跑步教练 App（2021 伦敦创立，2025-04 被 Strava 收购，估值传闻 ~£150M 未证实）。它是产品质量的基准，但**它自己进不来中国**——无中文、母公司 Strava 大陆受限、不接国产表。

**它强在哪（不要正面硬刚）**：

- **教练撰写模板 + 个性化引擎**（非生成式 AI），计划质量经验证、可信；
- **集成化力量 + 柔韧训练**（用户最称赞的差异点，第一卖点）；
- 打磨精良的 UX、广泛手表同步（结构化课程推到手表）、**Strava 分发 + 捆绑订阅**、Races by Runna 赛事发现；
- 定价对标真人教练极具性价比；App Store 4.9★（~3.1万评），180+ 国家 / 7 语言（含日语，**无中文**）。

**它的致命缺口（STRIDE 的突破口）**：

| Runna 缺口 | STRIDE 打法 |
|---|---|
| **不吃生理信号** —— 忽略心率/HRV/海拔/功率，**只改配速**，计划生成后即停滞，进阶跑者流失（DCRainmaker 实测）| **生理信号驱动的负荷/状态自适应 + 主观反馈闭环** —— 头号进攻点 |
| **无双向对话教练**，只有单向 insight | **对话即入口、双向教练** |
| **伤病处理是最大口碑负债**（默认增量过激，被指致伤）| 基于**运动医学知识库**的伤病应答 |
| 营养仅科普、装备推荐薄、**纯订阅无导购** | 完整训练服务（营养/恢复）+ **导购引擎** |
| **中国结构性缺位**：无中文、无华为/小米、无支付宝/微信支付、母公司 Strava 大陆受限、本土赛事稀薄 | **中国原生 everything + 多 OEM 设备（华为/小米/Apple/COROS/Garmin）** |

**结论**：Runna 的壁垒在于**集成化力量训练、成熟的产品体验与 Strava 生态分发**，这三点不宜正面强攻。针对 Runna 的**独有突破口**是它最薄弱处——**生理信号驱动的负荷/状态自适应**与**真正双向、具备伤病感知能力的对话式教练**（中国本土化与多设备覆盖是结构性背景，统一见末尾共性结论）。

### Sigma 深度分析（中国市场直接竞品之一）

Sigma（"你的 AI 跑步记录"，开发方**超超世世科技（上海）有限公司**）是目前国内**形态最接近 STRIDE 的中文 AI 跑步产品**，它的存在说明"中国市场无玩家"并不成立——但 Sigma 的重心与 STRIDE 不同，给我们留出了清晰的纵深差异。

- **产品特点**：由热爱跑步的**设计师团队**打造，强调"纯粹、仪式感"的体验——无打扰开跑、清爽的日/周/月多维分析、**地图日记**（记录跑过的城市与心情）、卡路里可视化；AI 伙伴接入**北京体育大学知识库**给个性化建议，可按心率实时提示降速；面向赛事的 **Sig AI 训练计划**（周期化、预测完赛成绩、生成配速策略）目前仍是**邀请制小范围尝鲜**。
- **市场/体量**：已上 App Store、小米应用商店、应用宝，小红书 @Sigma 运营；**公开订阅价与用户量暂未披露，整体处于早期**。
- **面向群体**：审美与记录驱动的**泛跑步 / 健康跑人群**，看重颜值、仪式感和轻量陪伴。
- **设备覆盖**：已支持苹果、佳明、华为、小米、高驰、华米、颂拓数据迁移——**中国设备生态覆盖与我们高度重合**，这一点不能再当作独占优势。
- **STRIDE 的机会**：Sigma 是**"记录 + 分析 + 轻 AI 伙伴"先行**、训练系统在后；周期化计划尚在早期邀请制，知识库偏**通识科普**而非个体生理建模。STRIDE 的纵深在于——**以训练系统为核心**（个体负荷模型 / PMC / 个性化基线 + 已落地的周期化执行闭环与持续跟踪）、**力量训练集成**、**伤病应答接运动医学知识**、以及**导购变现引擎**。即：不靠"也是中文"取胜，而靠**训练科学的深度与闭环**拉开差距。

### RQ（跑力 / RunningQuotient）深度分析（中国市场直接竞品之二）

RQ 是华语圈**训练科学最严谨**的数据分析型跑步产品（台北 PERPETUUM ONLINE 出品），在分析维度上与 STRIDE 的算法层最接近。

- **产品特点**：**重心是数据分析层**——以 **Jack Daniels VDOT「跑力值」** 为核心，做能力评估、最适配速区间、**训练压力指数（类 PMC/TSB）**、体能/疲劳/状态指标、各配速区间的跑姿技术分析。**训练计划是它的明显短板**：基于 VDOT 套出的**静态、写死的模板**，不随每日状态灵活调整，**也不实时追踪计划的执行情况**——本质是"看懂数据"的工具，而非"带你练"的闭环教练。
- **市场/体量**：台湾起家、华语严肃跑者中口碑扎实（"科学化训练"心智强）；订阅 US$12.99/月、$99.99/年。
- **面向群体**：**追求科学化训练的严肃跑者 / 教练**，看重数据严谨与方法论。
- **设备覆盖**：COROS / Garmin / Apple Watch——**不接华为/小米**，大陆主流国产表覆盖弱于 Sigma。
- **STRIDE 的机会**：RQ 强在"看懂数据 / 算跑力"，但**止步于分析层**——没有动态训练计划、没有执行追踪闭环；且繁体 / USD / 无国产表与本土支付，大陆本土化偏弱。STRIDE 把 RQ 级别的分析**接上对话式入口 + 真正逐日自适应的周期化执行闭环 + 国产表全覆盖的本土化**，从"分析工具"升级成"全程带练的教练"——这正是 RQ 缺的一段（通用差异化见末尾共性结论）。

### 其他 AI 跑步教练竞品

Runna 之外，还有三个**与 STRIDE 形态最接近**（AI 原生、计划自适应、订阅制）的产品值得深看——它们验证了"AI 教练"这条路走得通，但**体量都不大、面向欧美、均未本地化进入中国**：中国侧的真实竞争来自 Sigma / RQ，**不是它们**。

**1. TrainAsONE —— 算法最硬核的老牌玩家**

- **产品特点**：纯机器学习驱动，**每日重排计划**（run-by-run 自适应），主打**伤病预防**；会读完成度、缺训、病后恢复，甚至**接天气预报**调整当日课；支持任意距离（5K→多日超马）与多目标并行。
- **市场/体量**：品类里**入场最早**、算法口碑最深，但用户体量小众；累计分析 1 亿+ 公里跑步数据（官方宣称）。**被 COROS 收购**（约 2024-12，媒体报道、未独立核实 → 与下方 COROS Coach 同源整合中）。定价 £9.99/月、£99.99/年，有免费档。
- **面向群体**：**数据驱动型进阶跑者**，追求最深 AI 个性化、最在意防伤。
- **STRIDE 的机会**：它强在算法但 **UI/对话体验弱、无中文、被收编进 COROS 生态后偏硬件绑定**；我们用**对话式低门槛 + 中国本土化 + 多设备（不锁单一表厂）**错位竞争。

**2. Humango —— 多项目 + 对话教练 Hugo**

- **产品特点**：AI 教练 **Hugo** 提供对话式指导与计划动态调整；**多项目覆盖**（跑 / 骑 / 铁三），按赛事自动周期化。

**洞察**：很多跑者卡在"想要个性化专业指导，但所在城市没有合适教练、或通勤成本太高"。STRIDE 用 AI 把**个体数据驱动的专业指导**做到**随时随地可达、不依赖地理**——这是线下教练和群发训练营都做不到的格子。

**STRIDE 的楔子**：**AI 原生 × 专业训练科学 × 个体数据 × 中国本地化 × 对话式低门槛 × 随时随地可达**。

## 3. 核心交互：对话即入口

- 用户在**一个对话框**里说任意诉求（建计划 / 调计划 / 问状态 / 挪课 / 伤病 / 装备 / 选比赛 / 订酒店 …）。
- 系统识别意图，路由到对应能力作答或给出提案。
- 一条**滚动会话**贯穿连续旅程（选比赛 → 定赛事 → 排计划 → taper → 赛后复盘），会话记得住**当前操作对象**（目标赛事 / 训练计划 / 某一周），意图可逐轮切换。

### 两种交互模式

| 模式 | 谁发起 | 例子 |
|---|---|---|
| **被动响应** | 用户提问 | "我状态怎么样？""把明天强度课挪到今天" |
| **主动教练（push）** | 系统主动 | "今天该跑 8K 轻松跑了""HRV 连续走低，建议今天降强度""杭马报名明天截止""你的竞速鞋已跑 600K，该换了" |

主动教练是**留存与差异化的核心** —— 真教练会主动提醒、预警、督促，而不是干等用户来问。主动推送基于个体数据与计划进度触发，让 STRIDE 像"装在手机里、时刻在线的私人教练"。

### 交付形态

**App + Web 双端，App 优先**：

- **App**：主战场 —— 日常陪伴、主动推送、轻量对话、随时随地。覆盖最广跑者。
- **Web**：面向**资深 / 专业用户** —— 深度数据分析、计划全景查看与编辑、大屏复盘。

### 不变量（产品级约束）

1. **写操作先提案、后确认**：先产出提案卡片，用户确认才生效，绝不自动改动用户计划。
2. **安全敏感意图绝不自动改计划**：伤病、身体不适只给建议；降量/规避提案需用户明确同意。
3. **专业可信**：训练与伤病建议基于**自建专业知识库**（参考专业训练学与运动医学），不靠模型臆断 —— 这是订阅信任的根基。
4. **外部信息不编造**：比赛日历、装备型号、酒店等必须有真实数据源（厂家 / 合作方 / 自建库），不靠模型臆造。
5. **导购透明可信**：推荐基于个体数据适配，不是付费竞价排序优先；商业推荐与中立建议明确区分（信任是订阅留存的命）。

## 6. 战略分层 — 订阅命脉 vs 导购薄壳

> **第一原则：订阅护城河 = 训练服务的完整性。**
> 跑者的训练需求是一个整体 —— 练什么、怎么恢复、吃什么、伤了怎么办、天气变了怎么调。**只服务一半，跑者就会流失。** 唯有覆盖跑者的**全部训练需求**，才有源源不断的付费订阅。
> 因此：凡"训练相关"的能力域，**无论数据协同强弱，都是核心命脉**，都进订阅引擎 —— 营养、恢复、改课不是"加分项"，它们是训练的有机组成。

| 档 | 能力域 | 定位 | 变现引擎 |
|---|---|---|---|
| 🟢 **核心命脉 — 完整训练服务** | 0 计划 · 1 洞察 · 2 健康 · 3 营养 · 4 恢复 · 6 比赛策略 · 8 环境改课 · 10 知识 | 服务跑者**全部训练需求**，完整性本身就是留存护城河 | **订阅引擎 R1** |
| 🟡 **外部世界 — 导购薄壳** | 5 装备 · 6 比赛发现 · 7 酒店（+ 2/3/4 的实物导购，见下「内容 vs 实物」）| 接外部数据（厂家/合作方/自建库），用个体语境精准推荐 | **导购引擎 R2/R3/R4** |
| ⚪ **社区 — 引流** | 9 社交 | 复用社区基础，拉新与留存 | 引流 → 后续品牌/广告 |

**核心命脉内部的深度梯度**（是精细化，不是分层 —— 它们全是命脉）：

- **强数据驱动（★★★）**：0 计划 · 1 洞察 · 2 健康 · 6 策略 —— 算法精度 + 周期化闭环咬合最深，复制门槛最高（需数据量 + 工程 + 闭环三者兼备）。
- **知识 × 数据（★★☆）**：3 营养 · 4 恢复 · 8 改课 · 10 知识 —— 领域知识 + 个体数据个性化，比通用 LLM 强、且补全训练闭环。

**内容 vs 实物（营养 / 恢复为何横跨两引擎）**：2 健康 · 3 营养 · 4 恢复 的**建议性内容**（吃什么、怎么恢复、伤了怎么办）在订阅付费墙后，属核心命脉；其**实物导购**（gel / 电解质 / 泡沫轴 / 按摩 O2O）走导购引擎、对所有档开放。同一能力域横跨两引擎，是按"**内容订阅 vs 实物成交**"切分，并不矛盾。

**两引擎关系**：
- **订阅引擎**变现"完整训练服务"——覆盖越全，留存越牢，月费越值。
- **导购引擎**变现"外部世界"——天然按成交抽成；即使用户不订阅也能变现。
- 互为飞轮：订阅用户的个体数据让导购推荐更精准；导购触点又把免费用户拉向订阅。
- **6 比赛**横跨两边：**比赛策略**（配速/补给/taper）是订阅命脉，**比赛发现**（搜赛事）是导购薄壳。

## 7. 商业模式 — 双引擎

### 7.1 双引擎模型

```
引擎一：订阅（recurring，变现「完整训练服务」护城河）
   完整覆盖训练需求：规划 + 数据分析 + 健康/伤病 + 营养 + 恢复 + 改课 + 比赛策略 + 知识 → 月/年费
   留存逻辑：服务越全，跑者越离不开

引擎二：导购分成（transactional，变现「外部世界」）
   跑鞋/装备/补给 + 比赛发现报名 + 酒店旅行 → 按成交 CPS / 导流分成
```

两引擎共享同一资产：**个体数据**。它让订阅内容更专业、让导购推荐更精准 —— 数据越多，两边都更值钱（飞轮）。

### 7.2 收入流清单

| # | 收入流 | 类型 | 能力域 | 单位经济（假设，待验证）|
|---|---|---|---|---|
| R1 | **个人订阅** | 月/年费 recurring | 0,1,2,3,4,6,8,10 | **¥49/月**；含**跑步训练 + 力量训练**个性化指导；6 月付 **9 折**、年付 **8 折** |
| R2 | **装备导购分成** | CPS 佣金 | 5,3,4 | 跑鞋/装备联盟 5–15%；高客单竞速鞋 |
| R3 | **比赛报名导流** | 按单 / % | 6 | 固定 ¥/单 或报名费 % |
| R4 | **酒店/旅行分成** | OTA booking | 7 | 4–10% booking commission |
| R5 | **品牌合作 / 测评** | 赞助费 | 5,6 | 新品测评、品牌植入、官方训练伙伴 |
| R6 | **真人教练 marketplace** | 平台抽成 | 0,2 | 撮合**远程在线**真人教练（不受本地供给限制），抽 15–30% |

**主轴**：R1 订阅（稳定现金流，估值锚）+ R2 导购（高毛利，免费用户也能变现）。R5/R7 是规模化后的 B2B 放大器。

### 7.3 合作伙伴 & 数据反哺

- **装备厂家**：合作获取**官方产品库数据**（型号/参数/库存/价格）→ 喂给推荐引擎；按 CPS 分成。用户数据语境（步态/跑量/伤病）= 比厂家自有渠道更高转化，是谈判筹码。
- **赛事方**：成为官方训练计划合作伙伴；报名导流；赛道数据合作。
- **酒店/OTA**：比赛周后勤导流。
- **数据反哺品牌（R7）**：匿名聚合洞察（如"某价位竞速鞋在 sub-3.5 跑者中换鞋周期 X 公里"）反向赋能品牌选品 —— **必须隐私合规、用户授权、仅聚合不可逆**。

```
Free 引流：同步数据、看基础状态、有限对话、社区、装备/比赛被导购（R2/R3 已可变现）
   ↓ 钩子：专业分析预览、计划生成 teaser、"想看完整赛季规划？升级 Pro"
Pro 付费墙：完整对话 / 赛季计划 / 专业分析 / 健康伤病 / 营养 / 比赛策略
   ↓ 升级：备赛旺季 / 想要真人教练背书
Elite：真人教练 + 深度报告
```

**设计原则**：
- 免费档要**有用到肯留存**（数据同步 + 社区 + 导购），但**专业深度留在付费墙后**。
- 导购（R2/R3/R4）**对所有档开放** —— 免费用户也是变现资产。
- 付费钩子绑在**高意图时刻**：建赛季计划、深度诊断、比赛策略。

## 10. 护城河资产 — 四层飞轮（归一化基线 + 算法 + 知识 + 闭环）

原始数据是底料，**不是护城河**（见 §1.3）；护城河是其上四者咬合的飞轮：**① 多设备数据归一化工程基线（对标 RQ、非独有）** + **② 算法分析（衍生洞察）** + **③ 专业知识库** + **④ 周期化训练闭环**。下表标注每项属于哪层。

**多设备接入**：原始运动与健康数据覆盖主流来源 —— **华为运动健康、小米、Apple Watch、COROS、Garmin，以及纯手机 GPS** —— 不绑定单一手表品牌，最大化可触达跑者与冷启动导入。

| 资产 | 来源 | 护城河层 | 支撑能力域 | 商业价值 |
|---|---|---|---|---|
| 运动原始数据（配速/心率/分段/轨迹/时间序列）| 主流手表同步 + 手机 GPS | 底料 | 1,2,6 | 订阅深度 + 导购精准 |
| **多设备数据归一化（清洗/对齐/统一口径）** | 自研工程（对标 RQ、非独有）| **① 归一化基线层** | 全域 | 一切分析的地基；国产表覆盖更广 |
| 训练负荷模型（慢性/急性负荷·状态）| 公开公式 + 自研校准/工程 | **② 算法层** | 0,1,6 | 订阅核心 |
| 个体基线（最大心率/乳酸阈/阈值配速/心率区间）| 公开公式 + 自研校准/工程 | **② 算法层** | 0,1,2 | 个性化推荐基底 |
| 能力快照 / 瓶颈诊断 | 公开公式 + 自研校准/工程 | **② 算法层** | 1,2 | 专业洞察差异化 |
| **专业训练 & 运动医学知识库** | 自建（参考权威训练学/运动医学）| **知识层** | 2,3,4,10 | **可信 AI 的根基**（伤病/营养/恢复敢答）|
| **周期化训练计划 + 执行跟踪闭环** | 规划 → 执行 → 跟踪 → 调整 | **闭环层** | 0 | **订阅留存命脉**（越用越贴合）|
| 体测数据（体重/体脂/骨骼肌等）| 合作体测仪（如 InBody）/ 手动输入 | 底料 | 2,3 | 营养/减重订阅 |

| 社交信号（点赞/评论/关注）| 社区 | 旁支 | 9 | 社区引流 |
| 装备 / 步态语境（衍生）| 运动数据 + 基线 | 算法层 | 5 | **导购转化关键** |

## 11. 迭代路线（产品 + 商业化节奏，建议非承诺）

- **P0 统一对话入口**：一个对话框识别意图并路由，打通建计划 / 改计划 / 状态问答。｜商业：上线 **R1 订阅墙**（Free vs Pro）。
- **P1 训练核心补全**：挪课/跳课、feedback、改目标重排（域 0 闭环）。｜商业：转化漏斗调优。
- **P2 身体 & 安全**：伤病安全门控 + 重返跑步、力量诊断、睡眠/HRV 联动（域 2）。｜商业：Elite 档 + R6 真人教练试点。
- **P3 数据加持咨询**：营养、恢复、天气改课（域 3/4/8）。｜商业：**R2 补给/恢复品导购**接入。
- **P4 外部世界**：比赛发现+策略、装备、后勤（域 5/6/7，建外部数据接入层）。｜商业：**R2 跑鞋导购 + R3 报名 + R4 酒店 + R5 品牌合作**。
- **P5 社区 & 知识 & 数据产品**：社交扩展、知识库（订阅内含）、聚合数据（域 9/10）。｜商业：**R7 B2B 数据**。

## 12. 获客 / GTM（冷启动 + 增长）

### 12.1 破解冷启动 —— 手表数据一键导入

护城河越依赖数据，新用户**冷启动**越是命门。破法：onboarding 即同步主流手表（华为运动健康 / 小米 / Apple Watch / COROS / Garmin）的**历史数据**，结合用户填写的目标与 profile，**Day-1 就兑现价值** —— 给出"当前能力评估 + 目标可行性判断 + 首周计划"，无需漫长积累期。多设备覆盖既扩大可触达人群，也是获客的最低门槛入口。

### 12.2 获客渠道

| 渠道 | 打法 | 目标人群 |
|---|---|---|
| **手表用户导入** | 已有手表的跑者一键同步即见价值（最低门槛 + 解冷启动）| 全体有表跑者 |
| **内容 / KOL** | 小红书 / B站 / 抖音跑步博主测评 + 专业训练科普，建立专业信任 | 进阶 + 新手 |
| **赛事 / 跑团合作** | 与马拉松赛事、跑团合作，报名季精准触达 | 备赛跑者 |
| **社区裂变** | PB 庆祝、计划/成绩分享、约跑邀请，老带新 | 全体 |
| **应用商店 / SEO** | "跑步训练""马拉松计划""配速心率"等关键词 | 主动搜索者 |
| **免费增值漏斗** | Free 拉新 → 高意图时刻（建赛季计划/深度诊断）转 Pro | 全体 |

### 12.3 增长飞轮

```
更多用户 → 更多数据 → 算法与推荐更准 → 留存更高 + 口碑扩散 → 更多用户
```

数据飞轮同时强化两条护城河（算法 + 知识库个性化）与两个变现引擎（订阅留存 + 导购精准）。

## 13. 开放问题

1. **定价验证**：¥49/月（6 月 9 折 / 年付 8 折）是否匹配中国跑者付费意愿？Elite/备赛包形态与定价？
2. **外部数据选型**：赛事日历、跑鞋库、酒店各接谁（厂家直连 / 联盟平台 / 自建）？
3. **导购合规与信任**：商业推荐 vs 中立建议如何在对话里明示区分，避免侵蚀订阅信任？
4. **伤病医疗边界**：建议到哪一步为止（降量提示 vs 转诊），合规把关。
5. **数据隐私（R7）**：聚合数据 B2B 的用户授权、匿名化、合规框架。
6. **女性月经周期**：手表多无此数据 → 需手动录入 UI。
7. **遗漏边界**：跨项目（铁三：骑行/游泳）？数据导出/年度报告作为付费钩子？
