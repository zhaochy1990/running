name: weekly-key-session-skeleton
description: Compact per-week key-session skeleton rules for S1 master plans.
---
**Weekly key-session skeleton (HARD)**:
- `weeks` covers every active/planned natural week from first non-completed phase through `plan.end_date`; `week_index` +1, `week_start` Monday. Completed carry-over phases (`is_completed:true`) are timeline only: **emit no weeks**.
- Link `phase_name`; highs in phase range. Recovery=70-80% previous load (20-30% cut), not 70-80% of phase lower bound; avoid 40-50% cuts except taper/race.
- `key_sessions[]` only lists load/adaptation drivers: `long_run`, `threshold`, `tempo`, `interval`, `vo2max`, `hill`, `race_pace`, `time_trial`, `tune_up_race`, `race`, `strength_key`; omit easy/aerobic/recovery/commute runs.
- Session counts: load weeks 1-3 key sessions; recovery 0-1; race weeks may contain only one `race` and no `interval`/`vo2max`/`tempo`/`time_trial`/`tune_up_race`/`strength_key`; for 5K race week, mention strides/activation only in `focus`, `purpose`, or `coach_note`. If `profile.weekly_run_days_max <= 3`, max 2/week; otherwise max 3. Distance uses `distance_km`; time uses `duration_min`; hard-session cap <=2/week.
- Weekly high includes all distance key sessions. FM race `42.2km` cannot have race-week high `28`; use `42.2-45km`.
- If `profile.weekly_run_days_max` is 5-6, state 5-6 run days plus at least one true rest/mobility day; `零剂量<=2` alone is not enough. Keep hard runs <=2/week plus long run; embedded race_pace in `long_run` is same stimulus.
- Taper is distance-specific. Mark `is_taper_week=true`; first taper week drops `target_weekly_km_high` >=25% vs highest pre-taper load week, rounding down: peak `42km` needs first taper `<=31km` (not 32), `70km` needs `<=52km`. FM default ~14 days (21 days only with an explicit freshness/travel/injury reason); HM ~7 days max 10; 10K 3-7; 5K 3-5. A 12+ week 5K/10K/HM cycle still uses short race-specific taper, not automatic FM taper.
- Volume ramp compares adjacent **load weeks**; recovery/taper do not reset budget. High may rise <=10% vs previous load week. `60 -> 42 recovery -> 62` is fine; illegal: `60 -> recovery -> 70`, `64 -> 72`, `70 -> 80`, `72 -> 82`, `66 -> recovery -> 74`, `74 -> recovery -> 89`, `78 -> recovery -> 92`, `80 -> 90`. Round next load high **down**: `28 -> max 30`, `30 -> max 33`, `38 -> max 41`, `42 -> max 46`, `64 -> max 70/71` (not 72), `66 -> max 73` (not 74), `68 -> max 74`, `70 -> max 77/78` (not 80), `72 -> max 79/80` (not 82), `74 -> max 81` (not 82/89), `78 -> max 85`, `80 -> max 88/89` (not 90), `82 -> max 90` (not 91/92). `32/92` FM peak needs previous load week >=84km. If context names a recent actual week (e.g. last week 62km), use it as first active load anchor unless completed-history only.
- Recovery cadence: every 4th week `is_recovery_week=true`, high=70-80% previous load (`75 -> 56-60`, not `75 -> 40-45`). No 5 consecutive non-deload weeks with >=2 hard stimuli; 10K speed/build 2-3 load then recovery; hard test/tune-up reduces other hard sessions.
- Peak long-run floors: fm >=28km; hm >=18km; 10k >=10km; 5k >=6km. Long-run size follows target distance, history, ramp/share math, goal realism, and the load-estimator anchor; do not write fixed weekly-volume caps in the prompt. Stale race data still keeps FM >=28km unless severe mismatch downgrades; no recent race with history peak ~52km should use a protected FM rehearsal followed by recovery, and gate A by 10K/HM evidence. For aggressive FM, choose the largest rehearsal that fits history/ramp/share math and A gates. Severe low-volume mismatch may relax the FM floor and must state multi-cycle reality.
- Long-run load concentration: default non-recovery/non-taper `long_run.distance_km <= 0.35 * target_weekly_km_high`; the load-estimator also checks long-run dose share. Required highs: `22km` >=63, `24km` >=69, `26km` >=75, `28km` >=80, `30km` >=86, `31km` >=89, `32km` >=92; never output `22/62`, `24/68` or `32/90` for normal/full-volume runners. If unsupported, lower long run or raise high only when history/ramp/load estimator supports it. Exceptions: frequency-limited, injury-return, severe-goal-mismatch, borderline-aggressive low-history runners; state trade-off and do not inflate weekly high just for math.
- FM/HM pace specificity: build/peak progress goal-pace work inside/paired with long runs, not only separate short sessions. Peak targets: FM ~22-25km MP inside 30-34km; HM ~12-16km HMP inside 18-24km. MP/HMP ladder rises by 2-4km, never jump >5km; sub-3 FM should hit `8 -> 12 -> 16 -> 18 -> 20 -> 22(-24)km`. Deload interrupts calendar, not ladder.
- Represent FM/HM goal-pace weeks with `long_run` total + embedded `race_pace` km (`同次长跑内MP段`), unless `weekly_run_days_max <= 3` then embed pace in `long_run.purpose`. If FM long run >=28km or race_pace >=8km, add no threshold/tempo/interval/vo2max/hill/time_trial/tune_up_race; only optional `strength_key`.
- Peak rehearsal placement: single biggest goal-pace rehearsal lands race-3..4 weeks, followed by true recovery/deload; never final load week before taper. Next week: `is_recovery_week=true`, high ~60-70% of peak (normally <=60-64km after 89-92km), long_run <=18-22km, no threshold/tempo/interval/vo2max/race_pace.
- No stacked maximal rehearsals: two largest goal-pace long runs cannot be consecutive load weeks; separate by reduced/recovery week. Achilles/quad-risk FM: separate any 30km+ weeks with recovery or clearly lighter long run (<28km and no race_pace); do not stack 30->31->32.
- Deload before taper/test: never run 4+ load weeks into taper; after last recovery <=3 load weeks. Tune-up/test weeks count as load and need reduced/recovery after if hard. Post-recovery re-entry rebuilds rhythm: high <= previous load week and avoid `time_trial`/`tune_up_race`, except the single biggest FM rehearsal at race-3..4 weeks followed by recovery.
- Sparse-device advanced self-report: no `time_trial`/`test_run`/`tune_up_race` to re-prove credible PR/history; derive start and peak from stated recent training, frequency, and the load-estimator anchor if present.
- Peak volume risk: do not chase an unproven volume record. Use the load-estimator anchor, recent active-week dose/km, historical peak, ramp, injury, and goal-realism gates to decide whether planned load should hold, increase, or reduce. Advanced athletes with high actual history must not be forced down to ordinary templates.
- Holiday-race intermediate FM: Spring Festival travel + intermediate + peak~58 + realistic PB -> peak `65-72km` (not 60-64 unless current-load/injury). Max long run before travel; in travel window (e.g. 2/9-2/15) **no `long_run` key session**: short Z2 + short MP/strides, hotel treadmill/flat loops, and state gear+fuel packing (shoes, race kit, gels/sodium, familiar breakfast). Avoid steep/new routes; no catch-up volume. Because the normal 28km <=35% share math would require 80km, allow only one protected `28km / 72km` max rehearsal followed by recovery; other long runs follow share math (`24km` needs >=69, `25km` >=72) and avoid `26km` because it needs >=75 outside the travel-adjusted load budget.
- Frequency-limited load budget: if `profile.weekly_run_days_max <= 3`, peak high <=60km and normally 45-55km. For realistic FM with history peak ~38km and small PB ask, default to `45-48km`, not a flat `40-42km` peak. Do not inflate for 35% math; use one protected max rehearsal `26-28km / 45-48km`, and for this fixture exactly `28km / 48km` (not 26/27), followed by recovery/deload. Earlier long runs ~20-23km; avoid `24km` until high >=48km unless protected rehearsal. Do not create many consecutive weeks where long-run share exceeds 50%. Use 1 long run + 1 quality + 1 medium aerobic; when long run contains MP/goal-pace or is >=24km, it is the quality stimulus and no tempo/threshold/interval/vo2max/hill/tune_up_race. Non-run days are strength/mobility/rest only; do not write "3 runs plus short jog/easy run" or add short jogs beyond the declared run-day budget.
- Injury-return load budget: if injuries/training gaps indicate recent return (停训/PT/康复/复跑/rebuild) and history has `peak_weekly_km_in_window`, start near 70% pre-injury baseline (55-60km -> about 30-35km; not <30 unless current load forces it). Use 2-4w rebuild + rebuild/base total 8-10w before build/speed. Knee/patellar/Achilles return with prior peak ~55-60km cannot use 80km+ peaks. FM return with prior peak ~58km: only the single 28km max rehearsal week may reach `64-65km`; do not create both a 64km and a 65km high week; all other load weeks stay <=63km and recovery follows. Chronic-but-controlled history without interruption (e.g. `慢性，可控；无痛`) is not a return-to-injury template.
- Knee/patellar return strength safety: `training_principles` must explicitly ban deep squat, lunge/弓步, high box jump, plyometrics/jump drills, high-intensity hills. Use wall sit/Spanish squat, leg press, glute bridge, calf raise, hip stability, pain-free step-down. Pain NRS>=4/swelling/AM stiffness -> reduce load.
- Strength & durability track: every plan needs non-running durability at phase/principle/milestone level; it does not count toward key-session caps. Each phase's `key_session_types` includes strength/durability (e.g. `strength_key`), `training_principles` states focus + cadence, and one milestone has `type:"strength_test"`. In `weeks.key_sessions`, use `strength_key` only for rehab/test/phase anchors; do not list routine maintenance strength every week. If injury/weakness exists, name targeted rehab, not generic strength.
