---
name: per-phase-milestones
description: A quantifiable exit milestone for each S1 phase (anchored to the actual PB / body-composition baseline, bounded by the physiological improvement ceiling).
---
**Per-phase quantifiable milestone (HARD)**:
- Each phase should **attach at least 1 quantifiable exit goal** on `milestone_ids` where possible (`metric`+`target_value`+`comparator`), and **anchor it to the actual baseline numbers injected above** (the "actual personal best (PB)" line in the history summary, **not** "COROS fitness prediction"; if available, use the weight / body-fat from the "body-composition baseline"). **Do not** write a generic goal detached from the baseline, and **do not** treat the fitness prediction as the PB.
- The target value's improvement rate must fall within the physiological ceiling (derive it from the "single-cycle improvement upper thresholds" above; and since a single phase usually covers only part of the whole cycle, the improvement margin is smaller):
  - **speed phase**: a 5k speed goal `metric:"race_time_s_5k"` (or `race_time_s_10k`), `comparator:"<="`. A ~6-week speed cycle's 5k improvement of **≤ ~5-8%** is the ceiling, not the floor — set target_value as baseline PB × (1 − improvement rate); do not set fantasy values.
  - **base phase**: exit on aerobic / long-distance capacity (a distance goal of `type:"long_run"`, or — if a body-composition baseline exists — a weight milestone `metric:"weight_kg"`,`comparator:"<="`, derived from a healthy weight-loss rate of ~0.25-0.5 kg/week, with no extreme deficit).
  - **build / specific phase**: MP endurance / threshold exit. For **fm/hm goals set a goal-pace-embedded long-run milestone**: `type:"long_run"` (or `"race_pace"`) whose `target` text states the total long-run distance AND the goal-pace volume inside it (e.g. "32km 长距，含 22km 马拉松配速"), with the embedded goal-pace volume escalating toward race specificity per the weekly-skeleton rule — not a generic easy-distance goal. Alternatively/additionally a `metric:"race_time_s_10k"`,`comparator:"<="` threshold check; if a body-composition baseline exists, you may set a body-fat goal `metric:"body_fat_pct"`,`comparator:"<="` (again at a healthy rate).
  - **peak / taper phase**: exit on race readiness (`type:"race"` or a target-distance time-trial race_time milestone).
  - **Tune-up race over solo time-trial (fm/hm goals)**: a real race tests goal-pace durability under competition fatigue far better than a solo time trial. For fm/hm goals, prefer programming a **tune-up race** (`type:"tune_up_race"`, an hm or 10k) ~4-6 weeks before race day as the build→peak fitness gate, with a `race_time_s_hm` / `race_time_s_10k` target derived from the actual PB (within the physiological ceiling). Keep a solo time-trial only as the fallback when a tune-up race cannot be scheduled.
- **Performance milestones are primary; body-composition is secondary (HARD)**: every phase's **primary exit milestone must be a performance / distance / goal-pace checkpoint** (a `race_time_*`, `long_run` distance, or embedded-MP-volume goal) — a milestone is only useful if it is a meaningful step toward the race goal. Body-composition milestones are an **optional supplement**, subject to ALL of:
  - **Set a body-composition milestone only when a "body-composition baseline" exists** — without one, set only performance / distance milestones; do not fabricate weight / body-fat numbers.
  - **At most ONE** body-composition milestone in the entire plan, and it must **never replace** a phase's performance exit milestone (it is added alongside, not instead).
  - **No "maintain current value" milestones (HARD)**: a milestone's `target_value` must represent a **meaningful change toward the goal**. A body-fat or weight milestone whose target equals (or barely differs from) today's baseline (e.g. "body_fat ≤ 22.9% 保持" when the baseline already is 22.9%) is near-zero signal — **do not emit it**. The same applies to any metric: never set a target the athlete has already met.
- **Milestone cadence (HARD)**: milestones must form a **continuous checkpoint ladder** — one roughly every **2-3 weeks**, with **no gap longer than 3 weeks** between consecutive milestone dates across the whole plan. The escalating MP-endurance long runs (e.g. 30km/16km-MP, 31km/18-20km-MP) are the natural way to fill the build→peak stretch: surface the key ones as `long_run` milestones so the athlete always has a near-term performance target, rather than leaving a 4-week hole between the strength test and the tune-up race.
- **Durability milestone (HARD)**: attach at least one `type:"strength_test"` milestone for the strength & durability track (see the weekly-skeleton strength rule). When the injected continuity / injury context flags an active injury or durability weakness, this milestone must target the **specific** deficit (e.g. "无痛完成跟腱偏心负荷 3×/周，单腿提踵左右差 < 10%"), not a generic strength goal — durability is a prerequisite for the race goal, not optional.
