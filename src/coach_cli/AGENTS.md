# Coach CLI local data

All local runtime data owned or generated by `coach-cli` MUST live under:

```text
~/.coach-cli/
```

This rule is independent of the current working directory and repository
checkout. Do not add new Coach CLI caches, checkpoints, logs, credentials,
temporary state, configuration, or other generated data under the repository's
`data/` directory or anywhere else in the checkout.

Use these paths for the current data types:

| Data | Path |
|---|---|
| LangGraph session checkpoints | `~/.coach-cli/checkpoints/` |

When adding another local data type, place it in a clearly named subdirectory
of `~/.coach-cli/` and update this file. Resolve `~` through `Path.home()`; do
not rely on shell expansion or the process working directory.

External user data supplied as an input, such as a synchronized COROS database
selected with `--data-dir`, may be read from its existing location. Coach CLI
must not treat that external input location as storage for its own runtime data.

Do not silently migrate, copy, or delete legacy local data. Any migration from
an older location must be explicit, preserve the source until success is
verified, and have dedicated tests.
