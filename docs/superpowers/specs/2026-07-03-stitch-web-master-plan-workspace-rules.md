# Stitch Web Master Plan Workspace Rules Decision

**日期**: 2026-07-03  
**范围**: STRIDE Web 端 Stitch 设计稿补齐与 Review 规则  
**状态**: 决策记录  
**主规范**: `frontend/DESIGN.md`

本文件记录 2026-07-03 对 Web 端 Master Plan / Weekly Plan / Coach Agent 工作区规则的确认。完整且后续持续维护的主规范已经迁移到 `frontend/DESIGN.md`。

## 核心产品场景

后续 Web 端 Stitch 设计稿围绕四个核心场景补齐:

1. 新用户生成 Master Plan。
2. 老用户修改 Master Plan。
3. 用户有 Master Plan 后，对生成的本周课表进行调整。
4. 用户的日常问答，包括基本训练问题、疲劳状态、伤病 / 疼痛风险分流。

## 核心决策

- 日常 AI / Coach 交流使用两栏: 左侧导航，右侧聊天框。
- 涉及当前计划、Plan Review、Diff Review 时使用三栏: 左侧导航，中间 Review / 当前计划，右侧聊天框。
- 新用户创建计划先使用两栏；计划生成后先展示完成卡片，由用户点击进入三栏 Plan Review。
- 存量用户从 Master Plan 首页点击 `调整计划` 后立即进入三栏，中间先展示当前 Master Plan。
- 用户从本周课表页点击 `调整本周` 后立即进入三栏，中间先展示当前 Weekly Plan。
- 日常问答保持两栏；只有用户明确要修改 Master Plan 或 Weekly Plan 时，才进入三栏。
- Weekly Plan 调整不默认改 Master Plan；只有用户明确确认，才上升为 Master Plan 调整。
- 三栏模式下，最终确认 CTA 必须归属中栏，且只保留一个 `启用计划` 主按钮；右侧 Coach Chat 只负责解释、追问和接收反馈。

## 路由判断

边界场景按以下顺序判断:

1. 用户正在查看、审阅、修改或确认某个计划实体时，使用三栏。
2. 新用户尚未看到可审阅计划时，使用两栏。
3. 用户只是提问、解释、复盘或咨询时，使用两栏。
4. 用户明确修改目标、周期、阶段重点或比赛安排时，进入 Master Plan 三栏调整。
5. 用户明确修改本周训练日、单次训练、恢复安排或短期强度时，进入 Weekly Plan 三栏调整。
6. Coach 不能仅因为识别到风险就自动切换三栏；必须让用户理解并确认要调整哪一个计划实体。

## 补齐优先级

Stitch 设计稿补齐优先级:

1. 新用户 Master Plan 闭环: 信息收集、生成等待、完成提示、Plan Review、启用成功。
2. 存量用户 Master Plan 调整闭环: 当前计划、调整意图、生成等待、Diff Review、启用成功。
3. Weekly Plan 调整闭环: 当前周课表、调整意图、生成等待、Week Diff Review、启用成功。
4. 日常问答高频状态: 基本训练问题、疲劳状态、伤病疼痛、从问答升级到计划调整。

## 设计稿资产策略

每次补齐 Stitch 设计稿前，先检查当前项目已有页面，并按以下口径处理:

- `保留`: 已符合布局规则、用户任务清晰、CTA 归属正确。
- `微调`: 结构方向正确，但需要修正信息层级、CTA 权重、状态文案或栏位职责。
- `重画`: 页面违反两栏 / 三栏规则，或把计划 Review 与聊天任务混在一起。
- `新增`: 当前项目没有覆盖该状态，尤其是 Weekly Plan 调整和日常问答。

四个核心场景都需要覆盖入口态、工作态和决策态。入口态说明用户从哪里进入；工作态承载 Coach 收集信息、追问、生成、解释和反馈；决策态让用户在中栏 Review 中确认 Plan Review / Diff Review 后才生效。

## 日常问答升级规则

- 基本训练问题继续保持两栏，可以引用计划和训练数据，但不进入 Review。
- 疲劳 / 跑不动状态默认保持两栏，Coach 先做状态分流，再询问是否调整本周。
- 伤病 / 疼痛问题默认保持两栏，Coach 先做风险分流和保守建议，再询问是否临时调整训练。
- 短期训练避让进入 Weekly Plan 三栏调整。
- 长期周期降强度、目标比赛变化、阶段重排进入 Master Plan 三栏调整。

后续 Stitch 设计稿补齐和 Review 均以 `frontend/DESIGN.md` 为准。
## 流程进度位置

- 左侧导航栏只承载产品导航、计划入口、当前对象和版本入口，不放创建、生成、审阅或调整进度。
- 两栏和三栏中的流程进度统一放在 Coach Chat 输入框上方。
- 中栏负责计划内容、当前处理内容、Diff、风险和最终 CTA，不承载全局流程 Stepper。
- 右侧进度位可以展示 `创建进度`、`生成进度`、`审阅进度`、`调整进度`，但不能出现与 `启用计划` 并列的决策按钮。
